### 🧩 插件简介
**YuRandomTeleport** 是一款围绕“安全随机传送”打造的 Bukkit 插件，支持按世界配置传送半径、高度范围、安全落点检测、冷却时间、等待倒计时、移动打断、Vault 经济扣费、传送他人、世界黑白名单、自定义成功/失败指令与消息文件等功能。

它适合生存服资源世界、野外探索、活动世界、RPG 大地图、空岛拓展世界以及需要减少出生点拥挤的服务器使用。

### 🧭 功能特性
- `/yurtp` 主命令，默认别名 `/rtp`
- 支持随机传送自己，或由管理员传送指定玩家到指定世界
- 按世界独立配置半径、费用、高度和安全规则
- 安全落点检测，支持禁用水、岩浆、虚空和危险方块
- 传送等待倒计时，移动、受伤、交互等操作可打断
- 冷却时间与绕过权限
- 可选 Vault 经济扣费，支持世界独立价格
- 传送开始、成功、失败时执行自定义控制台命令
- 配置热重载、消息自定义、调试日志

### 🧱 兼容性说明

- Paper

- Youer

经济功能为可选功能，启用后需要安装 **Vault** 与任意兼容的经济插件，例如 EssentialsX、CMI、iConomy 等。

### 🧩 Plugin Introduction
**YuRandomTeleport** is a Bukkit random teleport plugin focused on safe destination finding. It supports per-world radius, height ranges, safety checks, cooldowns, teleport wait time, movement cancellation, optional Vault economy cost, teleporting other players, world access control, custom success/failure commands, and customizable messages.

It is suitable for survival resource worlds, exploration servers, event worlds, RPG maps, skyblock expansion worlds, and servers that need to reduce spawn-area crowding.

### 🧭 Features
- `/yurtp` main command with `/rtp` alias
- Randomly teleport yourself, or let admins teleport a target player to a target world
- Per-world radius, cost, height, and safety rules
- Safe location checks with controls for water, lava, void, and dangerous blocks
- Teleport wait countdown with movement, damage, and interaction cancellation
- Cooldowns and bypass permissions
- Optional Vault economy cost with per-world pricing
- Custom console commands on teleport start, success, and failure
- Config reload, customizable messages, and debug logs

### 🧱 Compatibility
YuRandomTeleport aims to support **Minecraft 1.8.x - 1.26.x** across multiple server cores and versions, including but not limited to:

- Paper
- Spigot
- Paper forks
- Youer
- Arclight
- Mohist
- Other Bukkit / Spigot / Paper-based server cores

Economy support is optional. If enabled, it requires **Vault** and a compatible economy plugin such as EssentialsX, CMI, or iConomy.

If you encounter compatibility issues, please report them in the QQ group or GitHub Issues and include your server core, Minecraft version, plugin version, target world configuration, error logs, and reproduction steps whenever possible.
